# 10. 성능 & 접근성 가이드

## 1. 성능

- 효과 모듈은 requestAnimationFrame 기반으로만 렌더링하고, 숨김 상태에서는 루프 일시 중지
- 파티클/메시 개수 상한 설정, 저사양 모드 토글 제공
- 텍스처 압축 및 이미지 WebP 사용
- 코드 스플리팅 + preloading 전략 (핵심 UI 우선)

## 2. 접근성 (A11y)

- 키보드 탐색 가능: 리스트, 버튼, 슬라이더에 tabindex/aria 적용
- 대비 준수: 네온 포인트 컬러 사용 시 WCAG AA 대비 유지
- 모션 민감 사용자 배려: `prefers-reduced-motion` 감지 후 애니메이션 강도 낮춤
- 스크린리더 레이블: 주요 인터랙션 요소에 `aria-label` 제공

## 3. 반응형

- 모바일/태블릿 레이아웃에서 컨트롤 패널 접기/펼치기
- 캔버스는 부모 컨테이너 비율 유지, DPR 클램프(1.0~1.5)
